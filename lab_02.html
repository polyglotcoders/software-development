lab 2 Variables

Goals

  • Start creating your own variables
  • Understand what variables are

A variable is a value that can change

Try to predict what lucy would be on each print

variables.py

lucy = 10 + 2 - 1
print("lucy variable value: {}".format(lucy))
lucy = 13 + 2 - 1
print("lucy variable value: {}".format(lucy))

Execute:

python variables.py

Output:

$ python variables.py


lucy variable value: 11

lucy variable value: 14

Python has untyped variables

  • In python, variables can be of ANY type and you don’t have to declare it.

This is what it looks like in C / C++, where you have to declare the type of variable

typed_variables.h

int lucy = 1;
string colm = "Hello";
float morsal = 3.705;

This is what it looks like in python

untyped_variables.py

lucy = 1
colm = "Hello"
morsal = 3.705

Unlike normal math equations, the “conclusion” is on the left side

In math we do something like 1 + 2 + 3 = answer in programming however, we set the answer to the equation. answer = 1 + 2 + 3 The lefthand side almost always contains only one word.

Name of the variable does not matter

A common mistake people often make is thinking that the name of the variable matters. it does not.

name_does_not_matter.py

harry = 1.385
one = "Harry"

Run this file and you will see that it won’t throw any errors

Execute:

python name_does_not_matter.py

Output:

$ python name_does_not_matter.py

Another common mistake people make is thinking that there has to be quotation marks to variables. Variables never have quotation marks. They aren’t strings! Their value could be strings, but they are not.

quotation_marks_around_vars.py

"one" = "Harry"

Execute:

python quotation_marks_around_vars.py

Output:

$ python quotation_marks_around_vars.py

File “quotation_marks_around_vars.py”, line 1
“one” = “Harry”

SyntaxError: can’t assign to literal

Advanced: Return multiple values in python.

Python is quite unique in that you can make two variables equal to two statements. An example of this would be a function that returns two values.

return_two_values.py

def fn_return_two():
  return "a", "b"

str_1, str_2 = fn_return_two()

print("This is the first value returned {}".format(str_1))
print("This is the second value returned {}".format(str_2))

Execute:

python return_two_values.py

Output:

$ python return_two_values.py


This is the first value returned a

This is the second value returned b
